# 智慧政企小程序首页改版 PRD

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 智慧政企小程序 |
| 文档类型 | PRD |
| 需求范围 | 首页、应用页、工单入口、常用功能规则、角色化关键指标、小程序顶部标题 |
| 参考材料 | `小程序改版.doc`、`智慧政企小程序首页.html` |
| 当前版本 | V1.0 |
| 编写日期 | 2026-06-10 |

## 1. 背景

现有企微小程序首页承载个人信息、经营指标、常用功能和应用入口等能力，但不同角色的核心关注点不同，原首页需要进一步提升角色化展示、功能触达效率和政企办公场景下的数据看板体验。

本次改版基于业务侧提供的《小程序改版.doc》和当前高保真 HTML 原型，重新定义首页信息架构、角色判断规则、关键指标展示方式、常用功能排序规则、应用页分组与工单入口展示。

## 2. 目标

1. 首页按用户角色自动展示不同关键指标，降低无关信息干扰。
2. 管理员角色突出经营收入、毛利、回款、欠费和客户规模。
3. 客户经理角色突出五级升降、预计提成、三效产能、当月发展结果和过程动作。
4. 其他角色不展示关键指标模块，重点展示个人信息和常用功能。
5. 常用功能支持固定配置位与动态计算位混合排序，并支持模糊搜索。
6. 应用页按业务域分组展示全部功能，支持搜索定位应用。
7. 工单入口继承原能力，并在底部导航展示待办数量。
8. 页面视觉保持政企、专业、轻量、现代，符合企业级移动办公产品质感。

## 3. 用户角色与展示优先级

### 3.1 角色类型

| 展示类型 | 角色名称 | 角色编码 | 展示内容 |
| --- | --- | --- | --- |
| 管理员首页 | 政企 BG 管理员 | 381 | 管理员关键指标 |
| 管理员首页 | 政企区分管理员 | 382 | 管理员关键指标 |
| 管理员首页 | 政企区分营服管理员 | 387 | 管理员关键指标 |
| 客户经理首页 | 政企区分客户经理 | 388 | 客户经理关键指标 |
| 其他角色首页 | 不满足以上条件 | - | 不展示关键指标 |

### 3.2 判断优先级

角色判断按以下顺序执行：

1. 如果用户角色包含 381、382、387 中任一角色，展示管理员首页。
2. 否则如果用户角色包含 388，展示客户经理首页。
3. 否则展示其他角色首页。

说明：HTML 原型中的“角色模拟”仅用于设计预览和验收演示。实际前后端开发与正式上线版本均不展示角色模拟模块，页面展示内容由后端返回的用户角色信息驱动。

## 4. 页面整体结构

### 4.1 底部导航

底部导航固定为 3 个，不保留“我的”菜单：

| 菜单 | 说明 |
| --- | --- |
| 首页 | 当前改版重点页面 |
| 应用 | 全量应用功能页 |
| 工单 | 继承原工单能力，当前作为入口占位并显示待办数 |

### 4.2 小程序顶部标题

最终落地到微信小程序时，页面顶部导航栏标题按当前底部 Tab 动态展示：

| 当前 Tab | 小程序顶部标题 |
| --- | --- |
| 首页 | 智慧政企 |
| 应用 | 功能应用 |
| 工单 | 我的工单 |

说明：这里指微信小程序原生导航栏标题，不是在页面内容区额外增加标题。应用页内容区仍不展示“工作台”等顶部大标题。

### 4.3 首页模块

首页从上到下包含：

1. 个人信息区。
2. 常用功能区。
3. 关键指标区，仅管理员和客户经理展示。

其他角色不展示关键指标模块，常用功能区域展示更多入口，使页面不显空。

## 5. 首页需求

### 5.1 个人信息区

展示字段：

| 字段 | 说明 |
| --- | --- |
| 企微头像 | 展示企微头像或头像占位图标 |
| 姓名 | 展示用户姓名 |
| 岗位 | 展示用户岗位标签 |
| 部门 | 展示用户所属部门 |
| 科室 | 展示用户所属科室 |
| 退出登录 | 轻量线性图标，位于姓名与岗位同一行 |

交互要求：

1. 点击头像或退出登录图标，弹出退出登录确认或提示。
2. 用户确认退出后清理登录态，并跳转至登录流程。
3. 退出图标不使用大面积边框或重按钮样式，保持轻量化。

### 5.2 管理员关键指标

展示条件：用户角色包含 381、382、387 中任一角色。

指标内容：

| 指标 | 展示形式 | 单位 | 交互 |
| --- | --- | --- | --- |
| 年度考核收入 | 环形进度指标 | 万元 | 暂不要求跳转 |
| 年度考核毛利 | 环形进度指标 | 万元 | 暂不要求跳转 |
| 本月回款 | 环形进度指标 | 万元 | 暂不要求跳转 |
| 实时欠费 | 数据卡片 | 万元 | 点击跳转至欠费总览 |
| 存量客户 | 数据卡片 | 户 | 点击跳转至要客深耕 |
| 年累突破客户 | 数据卡片 | 户 | 点击跳转至要客深耕 |

视觉要求：

1. 三个核心财务指标使用环形进度样式，颜色统一为偏科技蓝的高级低饱和色。
2. 实时欠费需要相对醒目，但避免高饱和警告色造成刺眼。
3. 卡片间距紧凑但不拥挤，保持数据看板感。

### 5.3 客户经理关键指标

展示条件：用户角色包含 388，且不满足管理员角色条件。

指标内容：

| 指标 | 展示形式 | 数据口径 | 交互 |
| --- | --- | --- | --- |
| 五级升降 | 横向信息条 | 取原小程序“我的”数据 | 点击前置条件或岗级弹框查看 |
| 预计提成 | 数据卡片 | 取【客户经理工作台】最新月份数据 | 点击跳转客户经理工作台 |
| 三效产能 | 数据卡片 | 取【客户经理工作台】最新月份数据 | 点击跳转客户经理工作台 |
| 当月指标-移网 | 紧凑指标卡片 | 取【客户经理工作台】发展产能中的移网数据 | 点击查看明细，支持导出选择的tab，导出文件名为:本月移网发展/本周移网发展 |
| 当月指标-双线 | 紧凑指标卡片 | 取【客户经理工作台】发展产能中的双线数据 | 点击查看明细，支持导出选择的tab，导出文件名为:本月双线发展/本月价值双线发展 |
| 当月指标-融合 | 紧凑指标卡片 | 取【客户经理工作台】发展产能中的融合数据 | 点击查看明细，支持导出选择的tab，导出文件名为:本月融合发展/本周融合发展 |
| 当月指标-报备商机 | 紧凑指标卡片 | 取客户经理本月报备商机数量 | 点击跳转至“我的商机” |
| 当月指标-客户拜访 | 紧凑指标卡片 | 取客户经理本月客户拜访次数 | 点击跳转至“我的拜访”，并携带参数：拜访类型=客户拜访 |
| 当月指标-物业拜访 | 紧凑指标卡片 | 取客户经理本月物业拜访次数 | 点击跳转至“我的拜访”，并携带参数：拜访类型=物业拜访 |
| 当月指标-展厅邀约 | 紧凑指标卡片 | 取客户经理本月展厅邀约次数 | 不支持下钻，不跳转 |

五级升降弹框要求：

1. 点击“前置条件”展示固网双线新发展、创新业务新发展等条件明细。
2. 点击“岗级”展示当前月均新增收入、组内排名、预计升降级结果。

三效产能要求：
若用户角色包含942，则隐藏三效产能卡片，将预计提成卡片拉伸至与五级升降等宽。

发展明细弹框要求：

1. 支持移网、双线、融合三类明细。
2. 支持按本月、本周或全部、价值双线等维度切换。
3. 明细字段包含产品号码、客户名称、产品名称，不展示发展人字段。
4. 支持导出明细。

当月指标展示要求：

1. “当月指标”作为轻量标题，仅展示一次，不在每个指标名称前重复展示“当月”。
2. 指标名称统一使用与“预计提成”“三效产能”一致的灰色标签样式。
3. 可下钻指标不展示箭头，通过数值颜色区分可点击状态；可下钻数值使用蓝色。
4. 不可下钻的“展厅邀约”数值使用灰色，不绑定点击事件。
5. 移网、双线、融合一行展示；报备商机、客户拜访、物业拜访、展厅邀约一行展示，保持首页一屏紧凑效果。

### 5.4 其他角色首页

展示条件：不满足管理员和客户经理角色。

展示内容：

1. 个人信息区。
2. 常用功能区。

不展示内容：

1. 不展示管理员关键指标。
2. 不展示客户经理关键指标。

版面要求：

其他角色常用功能区最多展示 12 个入口，页面保持完整，不因缺少关键指标而显得空。

## 6. 常用功能区

### 6.1 展示数量

| 用户类型 | 最大展示入口数 | 布局 |
| --- | --- | --- |
| 管理员 | 最多 8 个 | 2 行，每行 4 个，最后一个固定为全部 |
| 客户经理 | 最多 8 个 | 2 行，每行 4 个，最后一个固定为全部 |
| 其他角色 | 最多 12 个 | 3 行，每行 4 个，最后一个固定为全部 |

### 6.2 搜索

1. 常用功能区右侧提供搜索框。
2. 支持按功能名称模糊搜索。
3. 搜索无结果时展示空状态提示，并保留“全部”入口。

### 6.3 横向滚动

1. 常用功能支持横向滑动分页。
2. 每页按 4 列网格展示。
3. 多页时展示分页圆点。
4. “全部”入口固定出现在每页最后一个位置。

### 6.4 功能排序规则

常用功能由固定配置位和动态计算位组成。

固定配置位：

1. 最多 4 个。
2. 优先展示。
3. 支持通过字典或数据表配置。
4. 支持按角色配置数量和顺序。
5. 支持配置角色优先级，以兼容一个用户拥有多个角色的情况。

动态计算位：

1. 根据用户点击行为计算功能权重。
2. 权重按用户维度每日凌晨计算。
3. 前端按后端输出的动态位排序展示。

计分规则：

| 点击时间范围 | 单次得分 |
| --- | --- |
| 近 3 天 | 3 分 |
| 近 4-14 天 | 2 分 |
| 近 15-90 天 | 1 分 |

前端展示处理顺序：

1. 获取用户角色和用户可点击功能合集。
2. 获取该用户所属角色的固定配置位，排除不在可点击功能合集中的功能。
3. 获取该用户的动态计算位，排除不在可点击功能合集中的功能，并排除已在固定配置位中的功能。
4. 按固定配置位加动态计算位顺序展示。
5. 管理员和客户经理最多展示 8 个，其他角色最多展示 12 个。
6. 最后一个入口固定为“全部”。

特殊情况：

1. 新用户首次进入或老用户长期未登录时，动态位可能为空。
2. 如果功能数量小于等于 4 个，常用功能区高度缩减为 1 行。

## 8. 工单入口

1. 工单能力继承原功能。
2. 本次首页改版仅保留底部导航入口和占位展示。
3. 底部导航工单图标右上角展示待办数量。
4. 待办数量超过 99 时建议展示为 `99+`。
5. 进入工单 Tab 时，微信小程序原生导航栏标题显示为“我的工单”。

## 9. 交互流程

### 9.1 首页初始化

1. 用户打开小程序首页。
2. 前端请求用户信息、角色、权限功能合集、关键指标、常用功能排序。
3. 后端返回角色判断结果或角色编码集合。
4. 前端按角色优先级展示对应首页。
5. 加载常用功能和底部导航待办数。
6. 首页 Tab 对应微信小程序原生导航栏标题为“智慧政企”。

### 9.2 点击全部

1. 用户点击常用功能中的“全部”。
2. 页面切换到底部“应用”菜单。
3. 应用页展示全部应用分组。

### 9.3 点击管理员指标

1. 点击实时欠费，跳转欠费总览。
2. 点击存量客户，跳转要客深耕。
3. 点击年累突破客户，跳转要客深耕。

### 9.4 点击客户经理指标

1. 点击预计提成或三效产能，跳转客户经理工作台。
2. 点击前置条件，弹出前置条件明细。
3. 点击岗级，弹出绩效岗级明细。
4. 点击移网、双线、融合发展，弹出对应明细。
5. 点击报备商机，跳转至“我的商机”。
6. 点击客户拜访，跳转至“我的拜访”，并携带参数：`visitType=客户拜访`。
7. 点击物业拜访，跳转至“我的拜访”，并携带参数：`visitType=物业拜访`。
8. 展厅邀约不可点击，不触发下钻或跳转。
9. 点击导出，导出当前明细数据。

| customerVisitParams | 客户拜访跳转参数，需包含 `visitType=客户拜访` |
| propertyVisitRoute | 物业拜访跳转路由，目标为“我的拜访” |
| propertyVisitParams | 物业拜访跳转参数，需包含 `visitType=物业拜访` |
| showroomInviteClickable | 展厅邀约是否可点击，当前固定为 `false` |

### 10.6 埋点字段

常用功能、应用页功能入口、首页关键指标可跳转入口均需要做点击埋点。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| PAGE_NAME | string | 被点击入口对应的应用名称或目标页面名称 |
| PRE_PAGE_NAME | string | 点击来源，取值为：首页、应用、首页关键指标 |

取值规则：

| 点击来源 | PAGE_NAME | PRE_PAGE_NAME |
| --- | --- | --- |
| 首页常用功能 | 应用名称 | 首页 |
| 应用 Tab 功能列表 | 应用名称 | 应用 |
| 首页关键指标 | 指标对应目标应用名称 | 首页关键指标 |

示例：

1. 首页常用功能点击“商机报备”：`PAGE_NAME = 商机报备`，`PRE_PAGE_NAME = 首页`。
2. 应用页点击“欠费总览”：`PAGE_NAME = 欠费总览`，`PRE_PAGE_NAME = 应用`。
3. 首页关键指标点击“实时欠费”：`PAGE_NAME = 欠费总览`，`PRE_PAGE_NAME = 首页关键指标`。
4. 首页关键指标点击“存量客户”：`PAGE_NAME = 要客深耕`，`PRE_PAGE_NAME = 首页关键指标`。
5. 首页关键指标点击“报备商机”：`PAGE_NAME = 我的商机`，`PRE_PAGE_NAME = 首页关键指标`。
6. 首页关键指标点击“客户拜访”：`PAGE_NAME = 我的拜访`，`PRE_PAGE_NAME = 首页关键指标`，并在扩展参数中记录 `visitType = 客户拜访`。
7. 首页关键指标点击“物业拜访”：`PAGE_NAME = 我的拜访`，`PRE_PAGE_NAME = 首页关键指标`，并在扩展参数中记录 `visitType = 物业拜访`。

## 11. 视觉与体验要求

1. 页面背景采用浅灰或极浅蓝灰。
2. 卡片采用白底、轻阴影、圆角样式。
3. 主色使用联通红作为点缀，不大面积使用高饱和红。
4. 数据指标采用科技蓝、低饱和青绿、柔和金色、淡紫等体系。
5. 图标使用图形化图标，不使用中文文字代替图标。
6. 功能入口保持 4 列宫格，字号和间距适合移动端。
7. 底部导航固定在底部，符合小程序使用习惯。
8. 页面元素需适配安全区和主流 iPhone 机型。

## 12. 非功能需求

1. 首页首屏加载应尽量在 2 秒内完成。
2. 常用功能排序结果建议由后端每日凌晨预计算，避免前端计算压力。
3. 搜索输入应即时响应，不阻塞页面滚动。
4. 明细弹框表格支持横向滚动，避免字段挤压。
5. 所有跳转需要做权限校验，用户无权限时提示无访问权限。
6. 数据加载失败时展示兜底空状态或错误提示。

## 13. 验收标准

1. 管理员角色进入首页，能看到管理员 6 个关键指标。
2. 客户经理角色进入首页，能看到五级升降、预计提成、三效产能和当月指标。
3. 其他角色进入首页，不展示关键指标模块。
4. 点击“全部”能进入应用页。
5. 应用页按业绩看板、客户管理、提效工具分组展示。
6. 应用页不展示“工作台”顶部标题。
7. 首页、应用、工单三个 Tab 的小程序顶部标题分别为“智慧政企”“功能应用”“我的工单”。
8. 底部导航不展示“我的”菜单。
9. 常用功能支持模糊搜索。
10. 常用功能支持横向滑动分页。
11. 工单导航能展示待办数量。
12. 点击实时欠费跳转欠费总览。
13. 点击存量客户和年累突破客户跳转要客深耕。
14. 点击客户经理预计提成和三效产能跳转客户经理工作台。
15. 点击移网、双线、融合可查看明细并导出。
16. 点击报备商机跳转至我的商机。
17. 点击客户拜访跳转至我的拜访，并携带拜访类型为客户拜访。
18. 点击物业拜访跳转至我的拜访，并携带拜访类型为物业拜访。
19. 展厅邀约不支持下钻，不展示可点击态。
20. 点击头像或退出图标可触发退出登录流程。
